# 问题记录

1. 项目测试环境无法访问，正式环境正常，其他同测试环境的web项目都打不开，提示 `ERR_CONNECTION_REFUSED`

**背景**

刚把项目部署到测试环境，浏览器刷新后验证了部署内容。

接下来，先是本地服务提示 无法代理，以为是本地服务问题，终止重新启动还是一样不行。

想着通过 ssh 连接下测试环境，去看看是不是服务器出现问题了。报错，信息如下：

```sh
ssh:connect to host xxx.xx.xxx.xxx port xx: Connection refused
```

这下觉得是测试环境出错了，反馈 leader 后，他说他可以正常访问，也能连接服务器，其他同事也都可以，服务器上我的密钥也还在。

这下更奇怪了，尝试 `ping` 了下测试环境 web 域名，显示正常的没有丢包。

执行下列命令，返回同样的提示 `ERR_CONNECTION_REFUSED`。

```sh
curl  -vvv  https://xx.xxxxx.com
# curl  -vvv --noproxy "*" https://xx.xxxxx.com
```

这时看了下代理工具，关闭代理，访问出错。重启，访问出错。重新打开代理，换了个节点，就能正常访问了。

最终原因还是未知，怀疑是那个节点出了什么问题
