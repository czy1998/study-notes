# Jenkins

Jenkins 是一个开源的自动化服务器，最核心的作用是：

> 自动执行“构建、测试、部署”这一整套软件交付流程。

它属于：

- CI（Continuous Integration，持续集成）
- CD（Continuous Delivery / Deployment，持续交付/部署）

领域里最经典的工具之一。

你可以把它理解成：

> 一个“自动化流水线调度器”。

## 解决什么问题

没有它时我的开发流程：

```
开发写代码
↓
git push
↓
npm install
↓
npm run build
↓
手动 ssh 到服务器
```

push 之后的流程都需要手动操作

加入 `Jenkins` 后，流程变为：

```
开发写代码
↓
git push
```

后续的打包部署全部自动

## 核心作用

通常就四件事：

| 功能           | 作用              |
| -------------- | ----------------- |
| 持续集成（CI） | 自动构建+测试     |
| 持续部署（CD） | 自动上线          |
| 自动化任务     | 定时脚本/运维任务 |
| 流水线编排     | 串联复杂流程      |

## 核心概念

### 1. Job（任务）

最基础单位。

```
build-web
deploy-prod
run-test
```

### 2. Pipeline（流水线）

现代 Jenkins 核心。

把一系列步骤写成代码：

```
pipeline {
  stages {
    stage('Build') {
      steps {
        sh 'npm run build'
      }
    }

    stage('Test') {
      steps {
        sh 'npm test'
      }
    }
  }
}
```

这叫：

```
Jenkinsfile
```

本质：

> CI/CD 即代码（Pipeline as Code）

### 3. Agent / Node

真正执行任务的机器。

例如：

- 一台 Linux
- 一台 Mac
- Docker 容器

Jenkins master 调度它们干活。

### 4. Trigger（触发器）

什么时候运行。

比如：

| 触发方式    | 示例            |
| ----------- | --------------- |
| Git webhook | push 后自动运行 |
| 定时        | 每晚 1 点       |
| 手动        | 点按钮          |
| API         | 外部调用        |

## 实际案例

> 仓库内的 Jenkinsfile 文件 首字母要大写，不然会识别不到

这是一个前端 h5 项目内的 jenkinsfile

效果：

代码合并到 test 分支后 push，jenkins 定时检查分支，有新提交则在开始执行打包部署，打包成功/失败，都会通过 企业微信bot 发送消息进行通知

```
pipeline {
    agent any

    tools {
        nodejs 'node'
    }

    environment {
        PATH = "${WORKSPACE}/node_modules/.bin:${env.PATH}"
        HOME = '/var/jenkins_home'
    }

    stages {
        stage('Install') {
            steps {
                sh '''
                    set -e
                    node -v
                    npm install
                '''
            }
        }
        stage('Deploy') {
            steps {
                sh '''
                    set -e
                    npm run deploy:test
                '''
            }
        }
    }

    post {
        success {
            qyWechatNotification mentionedId: '', mentionedMobile: '', moreInfo: '', webhookUrl: 'https://qyapi.weixin.qq.com/cgi-bin/webhook/send?key=xxxxxxxxxx'
        }

        failure {
           qyWechatNotification mentionedId: '', mentionedMobile: '', moreInfo: '', webhookUrl: 'https://qyapi.weixin.qq.com/cgi-bin/webhook/send?key=xxxxxxxxxx'
        }
    }
}


```
