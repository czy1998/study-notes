# 自动化测试

## 如何使用

这里以[官方 demo](https://github.com/wechat-miniprogram/miniprogram-demo) 为例

1. git clone 项目，用 vscode 打开
2. 终端执行 `npm run init` 和 `npm i`
3. 使用 `微信开发者工具` 打开 `./miniprogram`
4. 开发者工具准备如下:
   - 编辑器设置界面-安全tab：打开 `CLI/HTTP 调用功能`
   - 工具-npm构建：触发构建
5. `./test/index.spec.js` 测试文件内容改为如下

   ```js
   const automator = require("miniprogram-automator");
   const { resolve } = require("node:path");

   describe("index", () => {
     let miniProgram;
     let page;

     beforeAll(async () => {
       miniProgram = await automator.launch({
         cliPath: "/Applications/微信开发者工具 2.app/Contents/MacOS/cli",
         // 开发着工具 的 cli 位置，如果你没有更改过默认安装位置，可以忽略此项
         projectPath: resolve(__dirname, "../miniprogram"), // 项目文件地址
       });
     }, 30000);

     it("desc", async () => {
       page = await miniProgram.reLaunch("/page/component/index");
       await page.waitFor(1000);
       const desc = await page.$(".index-desc");
       expect(desc.tagName).toBe("text");
       expect(await desc.text()).toContain("以下将展示小程序官方组件能力");
     }, 30000);
     // 由于测试环节本地执行时间超过 默认的限制 5s，故此改为 30s

     afterAll(async () => {
       await miniProgram.close();
     });
   });
   ```

6. package.json 中写入脚本 `"test": "jest ./test/index.spec.js"`
7. 终端执行 `npm run test`，输出如下

   ```sh

   > jest ./test/index.spec.js
   ```
