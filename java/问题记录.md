# 问题记录

1. 接口 A 执行的查询 sql 里是固定表名，通过接口调用可以查到数据，但是单独拿 sql 执行却查不到数据

   **原因**

   sql 里的是逻辑表名，实际可能存在分表，需要获取分表的表名，再去执行 sql

   **解决办法**
   1. 执行寻找工具方法，获取分表名称
   2. 替换 sql 里的分表名称

2. 接口 B 执行的查询 sql 里包含了 tb.attachment_list 映射类里也有 attachmentList 字段，返回结果里包含 attachmentList 字段。但是我单独查表 attachmentList 字段是有值的，
   但是接口执行后，其他同表字段都查出来了，就 attachmentList 字段返回空

   **原因**

   数据库中 attachment_list 的类型是字符串
   映射类里 attachmentList `List<String>` 字符串数组
   两者无法直接映射

   **解决办法**
   1. 映射类里添加 attachmentListStr 作为中间字段来转换

   ```java
    @JsonIgnore // 接口不会返回这个字段
    private String attachmentListStr;


    public void setAttachmentListStr(String attachmentListStr) {
        if (StringUtils.isNotBlank(attachmentListStr)) {
            this.attachmentList = JSON.parseArray(attachmentListStr, String.class);;
        }
    }
   ```

   2. resultMap 里指定 json

   ```xml
   <resultMap id="visit" type="com.visit.models.Visit">
        <id column="id" property="id"/>
        <result column="distance" property="distance"/>
        <result column="attachment_list" property="attachmentList"
                typeHandler="com.hooshine.hera.mybatis.JsonTypeHandler"/>
    </resultMap>
    <select id="getByCompanyId" resultMap="visit">
        SELECT
        <include refid="baseColumns"/>
        FROM tb_visit
        WHERE id = #{id}
        <if test="companyId != null">
            AND company_id = #{companyId}
        </if>
    </select>
   ```
